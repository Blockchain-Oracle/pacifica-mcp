# MCP Client Setup

Pacifica MCP works with **12+ MCP clients** out of the box. For quick setup with tabs for every client, see the [Installation guide](/getting-started/installation).

This page covers advanced configuration, environment variables, and troubleshooting.

---

## Quick reference

| Client | Install command |
|---|---|
| Claude Code | `claude mcp add pacifica npx @pacifica-dev/mcp` |
| Gemini CLI | `gemini mcp add pacifica npx -y @pacifica-dev/mcp` |
| Codex | `codex mcp add pacifica npx -y @pacifica-dev/mcp` |
| OpenClaw | `openclaw mcp set pacifica '{"command":"npx","args":["-y","@pacifica-dev/mcp"]}'` |
| Goose | `goose configure` → Add Extension → `npx -y @pacifica-dev/mcp` |

For **Claude Desktop, Cursor, Windsurf, VS Code, Zed, Continue, Cline** — add a JSON config entry. See the [Installation tabs](/getting-started/installation) for the exact config for each client.

The standard config pattern is always:

```json
{
  "mcpServers": {
    "pacifica": {
      "command": "npx",
      "args": ["-y", "@pacifica-dev/mcp"]
    }
  }
}
```

## Environment variables

| Variable | Default | Description |
|---|---|---|
| `PACIFICA_NETWORK` | `testnet` | `testnet` or `mainnet` |
| `PACIFICA_PRIVATE_KEY` | (auto-generated) | Base58-encoded Ed25519 private key override |
| `LOG_LEVEL` | `info` | Pino log level: `debug`, `info`, `warn`, `error` |

### Setting env vars

Add an `env` block to your MCP config:

```json
{
  "mcpServers": {
    "pacifica": {
      "command": "npx",
      "args": ["-y", "@pacifica-dev/mcp"],
      "env": {
        "PACIFICA_NETWORK": "mainnet"
      }
    }
  }
}
```

Or for Claude Code via CLI:

```bash
claude mcp add pacifica -e PACIFICA_NETWORK=mainnet -- npx @pacifica-dev/mcp
```

### Using an existing wallet

If you already have a Solana wallet and want to use it instead of the auto-generated one:

```json
{
  "env": {
    "PACIFICA_PRIVATE_KEY": "your-base58-encoded-ed25519-private-key"
  }
}
```

<Callout type="warning">
Never share your private key. The key stays local and is only used to sign requests.
</Callout>

## Verify the connection

After setup, check that the MCP server is running:

**Claude Code:**
```bash
claude mcp list
```

**OpenClaw:**
```
List all Pacifica tools
```

**Any client:** Ask the agent *"Show me my Pacifica wallet address"* — if it calls `pacifica-wallet` and returns an address, the server is connected.

## Troubleshooting

**"npx: command not found"**
Install Node.js 18+ from [nodejs.org](https://nodejs.org).

**Wallet not found**
The wallet config at `~/.pacifica-mcp/config.json` is created automatically on first use. Run any Pacifica tool to trigger wallet generation.

**"Request failed: 401" or "Account not found"**
Your wallet has no funds on Pacifica. Deposit funds (SOL, USDC, or other supported assets) via the Pacifica web app. See the [Testnet guide](/guides/testnet).

**Server not appearing in client**
Check that the JSON config is valid (no trailing commas) and restart the client after saving.

**Stale data after trading**
Pacifica MCP caches read-only responses for 5 minutes. After placing orders or changing settings, the cache is automatically cleared so subsequent reads return fresh data.
